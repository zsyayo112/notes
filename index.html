📦§1 Shell 变量与命令替换
变量赋值

格式：name=value，等号两边不能有空格。所有变量都是字符串类型。未初始化的变量值为空字符串。

变量引用

$name 或 ${name} 读取变量值。${name}suffix 可避免歧义，例如 ${x}y。

命令替换

$(command) 执行命令并捕获其 stdout（去掉末尾换行）。老写法：反引号 `` `command` ``，嵌套时有问题，建议用 $()。

算术展开

$((expression)) 进行 C 风格整数运算。变量前的 $ 可省略。例：x=$((x + 1))。

EXAMPLE — 变量与命令替换
$ x=8
$ answer=$((x*x - 3*x + 2))
$ echo $answer
42

$ now="$(date)"
$ echo "$now"
Fri 13 Mar 2026 15:29:25 AEDT

$ v=""
$ echo $v    # 空，无输出
$ echo "$v"  # 保留空字符串作为参数
⚠ 变量作用域陷阱 管道中的变量赋值在管道外无效！每个管道组件在子shell中运行。同理，脚本中 cd 或变量赋值对父shell无效——这正是 Week04 第一题的根因。

引号类型示例变量展开命令替换GlobbingWord Split

双引号 ""echo "$x *.c"✅✅❌❌

EXAMPLE — 引号对比
$ answer=42

The answer is $answer.

🔧§3 内置变量与参数
变量含义示例
$0脚本名称（命令名）echo $0 → ./myscript.sh
$1 $2 …第1, 2, …个命令行参数echo $1
$#参数个数test $# -eq 2

ℹ 退出状态约定 0 = 成功 / 正常；非0 = 失败 / 异常。/bin/true 始终返回0，/bin/false 始终返回1。Shell 的 if/while 都基于退出状态判断。

🧩§7 Shell 函数
函数语法与 local 变量
#!/bin/dash
is_prime() {
    local n i        # local 限制作用域
    n=$1
    i=2
    while test $i -lt $n
    do
        test $((n % i)) -eq 0 && return 1
        i=$((i + 1))
    done
    return 0
}

is_prime 7 && echo "7 is prime"
⚠ local 作用域 不用 local 声明的变量是全局的！函数里的 i 会覆盖外面的 i，破坏外层循环。exit 在函数内也会终止整个脚本，用 return 代替。

🗂§10 PATH 搜索机制
PATH 工作原理
$ echo $PATH
/usr/local/bin:/usr/bin:/bin:/home/z1234567/bin

# 输入 cat 时，shell 依次尝试：
#   /usr/local/bin/cat
#   /usr/bin/cat
#   /bin/cat  ← 找到了！

# 搜索 PATH 用 IFS=: 分割，然后对每个目录 stat
# which.sh 练习就是模拟这个过程

# 危险：. 在 PATH 开头
$ PATH=.:/bin:/usr/bin   # 当前目录优先 → 安全风险！

# 在脚本中固定 PATH（推荐）
PATH=/usr/local/bin:/usr/bin:/bin
🔴 安全警告 不要将 . 放在 PATH 开头（尤其是 root 用户）。攻击者可以在当前目录放一个同名恶意脚本来劫持你的命令。

Q1: start_lab04.sh 为什么不起作用？ 脚本中有 cd ~/labs/04 和变量赋值，但运行后 pwd 没变，echo $ex1 也没有输出。请解释原因并给出修复方法。
▶

# 原因：脚本在子Shell中运行
# 脚本中的 cd 和变量赋值只影响子Shell，父Shell不受影响

$ source ./start_lab04.sh

# 解决方法2：把脚本改成输出 export 语句，再 eval
# 这两种方法让命令在当前Shell上下文中执行
📖 要点

Shell 脚本默认在子进程中运行。子进程的环境变量和工作目录改变不会传播回父进程。只有 source（或 .）能让脚本在当前shell中执行。

Q3: Shruti 的符号链接为什么是坏链接？ for image_file in $(ls ~/friends); do ln -s "~/friends/$image_file" .; done
创建的链接全是坏链接，为什么？如何修复？
▶

# 问题1："~/friends/..." 在双引号内，~ 不展开！

~/foo    ← 不展开
$ echo ~/foo
/home/z1234567/foo  ← 正确展开

Q5: update_course_code.sh 写一个脚本，将给定文件中的 COMP2041→COMP2042 和 COMP9044→COMP9042。再扩展：如果参数是目录，递归处理该目录下所有文件。
▶
✅ 答案
#!/bin/dash
# update_course_code.sh

update_file() {
    sed -i 's/COMP2041/COMP2042/g; s/COMP9044/COMP9042/g' "$1"
    echo "Updated: $1"
}

for arg in "$@"
do
    if test -d "$arg"
    then
        # 递归处理目录下所有普通文件
        find "$arg" -type f | while read -r file
        do
            update_file "$file"
        done
    elif test -f "$arg"
    then
        update_file "$arg"
    else
        echo "$0: '$arg' not found" 1>&2
    fi
done
📖 关键点

sed -i 原地修改文件。find -type f 递归找普通文件。注意 while read -r file 比 $(ls) 更安全（处理含空格的文件名）。

Q7: which.sh 用 ls 和 $PATH 实现 which.sh program_name：在 PATH 的各目录中查找可执行文件并打印详细信息，未找到则打印 "lost not found"。
▶
✅ 答案
#!/bin/dash
program=$1
found=0

# 用 tr 将 PATH 中的 : 替换为换行，逐行读取目录
echo "$PATH" | tr ':' '\n' | while IFS= read -r dir
do
    fullpath="${dir}/${program}"
    if test -f "$fullpath" -a -x "$fullpath"
    then
        ls -l "$fullpath"
        found=1
        break
    fi
done

# 注意：break 无法跨越管道子shell
# 更简洁的版本：
for dir in $(echo "$PATH" | tr ':' ' ')
do
    if test -x "${dir}/${program}"
    then
        ls -l "${dir}/${program}"
        exit 0
    fi
done
echo "$program not found"
exit 1

Q8: recompile.sh（stat 版） 写 recompile.sh main.c：每当文件修改时间改变，就重新编译并运行程序。使用 stat -c '%Y' file 获取修改时间戳。
▶
✅ 答案
#!/bin/dash
src=$1
last_mtime=""

while true
do
    mtime=$(stat -c '%Y' "$src" 2>/dev/null)
    if test "$mtime" != "$last_mtime"
    then
        last_mtime=$mtime
        echo "Recompiling $src ..."
        if gcc "$src" -o a.out
        then
            ./a.out
        fi
    fi
    sleep 1
done

Q13: 子Shell陷阱修复 下面脚本想统计文件所有数字之和，但输出总是 0，为什么？给出两种修复方法。 
total=0
cat numbers.txt | while read n; do
    total=$((total + n))
done
echo "Total: $total"
▶
✅ 答案
# 原因：管道右边的 while 在子shell里运行
# 子shell对 total 的修改不影响父shell

# 修复方法1：重定向代替管道（避免子shell）
total=0
while read -r n
do
    total=$((total + n))
done <numbers.txt
echo "Total: $total"

# 修复方法2：用 awk（根本不需要shell变量）
awk '{sum += $1} END {print "Total:", sum}' numbers.txt

# 修复方法3：在子shell结束前输出结果
cat numbers.txt | { 
    total=0
    while read -r n; do total=$((total+n)); done
    echo "Total: $total"
}

is_prime_bad() {

is_prime() {
    local n i   # 关键！
    n=$1; i=2
    while test $i -lt $n; do
        test $((n % i)) -eq 0 && return 1
        i=$((i + 1))
    done
    return 0
}
i=2
while test $i -lt 20; do
    is_prime $i && echo "$i"
    i=$((i + 1))
done

Q16: 变量展开扩展语法 解释并举例以下语法：${var:-default}、${var:=default}、${var:?error}。在什么场景下会用到这些？
▶
✅ 答案
# ${var:-default}：var 未设置或为空时，使用 default（不赋值）
$ unset name; echo "${name:-stranger}"
stranger

# ${var:=default}：var 未设置或为空时，赋值 default 并使用
$ unset count; echo "${count:=0}"; echo $count
0
0  # count 被赋值了

# ${var:?message}：var 未设置时，打印 message 并退出脚本
$ unset required; echo "${required:?'ERROR: required not set'}"
bash: required: ERROR: required not set

# 实用场景：
# 脚本开头设置默认值
OUTPUT_DIR="${1:-/tmp/output}"
MAX="${MAX:=100}"
# 强制要求参数
API_KEY="${API_KEY:?'ERROR: API_KEY environment variable must be set'}"

Q17: 进程并行与 wait 写一个脚本，并行编译所有传入的 .c 文件（每个文件启动一个后台进程），所有编译完成后链接成一个可执行文件。

#!/bin/dash
# parallel_compile.sh file1.c file2.c ...

for f in "$@"
do
    gcc -c "$f" &    # & 表示后台运行
done

wait    # 等待所有后台进程完成

# 检查是否所有 .o 文件都生成了
if gcc -o program *.o 2>/dev/null
then
    echo "Build successful: ./program"
else
    echo "Link failed" 1>&2
    exit 1
fi

# wait 等待当前shell所有后台子进程完成

综合挑战：safe_process.sh

写一个脚本 safe_process.sh，要求：

1. 参数检查：接受至少一个目录名作为参数，否则打印 usage 到 stderr 并退出1
2. 函数：定义 count_lines(file) 函数（用 local）返回文件行数
3. 临时文件：用 mktemp 创建临时文件，trap 确保退出时清理
4. 递归遍历：用 find 遍历所有参数目录中的 .sh 文件
5. 统计：对每个 .sh 文件，用 is_business_hours 判断当前是否为工作时间，若是则统计该文件行数写入临时文件
6. 报告：最后用 sort -rn 输出"行数最多的前5个 shell 脚本"
7. 退出状态：如果没有找到任何 .sh 文件，退出1；否则退出0

串联知识点：参数处理 / 函数+local / mktemp+trap / find / 条件判断 / 管道 / sort / 退出状态
▶
✅ 参考答案
safe_process.sh
#!/bin/dash
# safe_process.sh - 综合挑战参考答案

# ── 1. 参数检查 ──────────────────────────────
if test "$#" -lt 1
then
    echo "Usage: $0 <directory> [directory...]" 1>&2
    exit 1
fi

# ── 2. 函数（带 local）───────────────────────
count_lines() {
    local file result
    file=$1
    result=$(wc -l <"$file")
    echo "$result"
}

# ── 3. 临时文件 + trap ────────────────────────
TMP=$(mktemp)
trap 'rm -f "$TMP"; exit' INT TERM EXIT

# ── 4. is_business_hours 内联版 ──────────────
is_business_hours() {
    local hour
    hour=$(date | cut -d' ' -f5 | cut -d':' -f1)
    test "$hour" -ge 9 -a "$hour" -lt 17
}

# ── 5. 遍历 .sh 文件 + 统计 ──────────────────
found=0
for dir in "$@"
do
    if ! test -d "$dir"
    then
        echo "$0: '$dir' is not a directory" 1>&2
        continue
    fi

    find "$dir" -name '*.sh' -type f | while IFS= read -r shfile
    do
        found=1
        if is_business_hours
        then
            lines=$(count_lines "$shfile")
            echo "$lines $shfile" >>"$TMP"
        fi
    done
done

# ── 6. 报告：行数最多的前5个 ─────────────────
if test -s "$TMP"
then
    echo "=== Top 5 Shell Scripts by Line Count ==="
    sort -rn "$TMP" | head -5
else
    echo "No .sh files processed (outside business hours or none found)"
fi

# ── 7. 退出状态 ───────────────────────────────
# 注意：find|while 在子shell中执行，found 变量改变不传出
# 改用 test -s 检查临时文件是否有内容作为替代
if ! test -s "$TMP"
then
    exit 1
fi
exit 0
📖 关键知识点回顾
✅ 参数检查$# 计数，1>&2 输出到 stderr
✅ local 变量函数内用 local 防止全局污染
✅ trap + mktemp安全临时文件，任何退出都清理
✅ find + while read安全遍历，正确处理含空格文件名
⚠ 管道子Shell陷阱find|while 子shell中变量赋值不传出，用 test -s 检测文件内容代替
ℹ 退出状态0=成功，非0=失败。test -s file 判断文件非空
